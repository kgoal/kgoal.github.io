Body 注解的坑

如果直接使用 @Body 注解来进行请求，例如如下 API 接口

这样子的写法是会报错的，Unable to create @Body converter for class LoginModel。这是因为在创建 Retrofit 实例时，没有制定特定的转换器，所以会去使用默认的转换器 BuiltlnConverters

因为默认的 converter 只处理 ResponseBody 类型的数据，所以在在指定非 ResponseBody 数据时，会返回 null，所以导致最终没有结果而抛出错误。

所以解决办法就是在创建 Retrofit 实例时，通过 addConverterFactory 添加特定的转换器，比如 Gson，这样就可以解决这个问题了。

官方文档对这个问题也有提到： By default, Retrofit can only deserialize HTTP bodies into OkHttp’s ResponseBody type and it can only accept its RequestBody type for @Body.
