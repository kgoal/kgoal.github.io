该方法设置了 mCancelled 状态，方便在 onCancelled 方法中处理取消后的业务。而 mFuture.cancel 会设置任务的状态为 onCancelled，并且调用到 done 方法，也就是在构造函数中进行重写的方法

在该方法中，会执行 postResultIfNotInvoked 方法，代码如下

获取的是任务是否被调用的判定值，如果此时任务未被调用，则会调用 postResult 的方法。而此时 AsyncTask 的 mCancelled 为 true，那么就会执行 onCancelled 的回调方法

该方法的存在意义就是在于，如果在任务未正式启动（被线程池执行）时，就已经取消了，那么此时的 mWorker 就不会被调用，那么就不会按照正常流程走到 postResult 方法，也就不会调用到 onCancelled 回调。所以存在该方法，来处理这个问题

总的来说，使用 FutureTask 可以更好的管理任务的状态
